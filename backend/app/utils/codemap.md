# backend/app/utils/

## Responsibility

- Houses cross-cutting helpers that keep Flask services thin: a shared exception taxonomy, idempotency guardrails, webhook validation, PDF/layout helpers, building-code lookups, and timezone utilities are all centralized here so the controllers and service layers can rely on a single source of truth.
- Keeps auxiliary models (`TeamsNotification`, ArcGIS building features, SharePoint artifacts) consistent by exposing small APIs that encode the real-world constraints (e.g., duplicate notification suppression, building-code extraction heuristics, CST-based scheduling).

## Design

- **Exception taxonomy:** `DNSApiError` is the base for all HTTP/JSON-friendly failures. Each subclass (`ValidationError`, `NotFoundError`, `StatusTransitionError`, `FileOperationError`, `ExternalServiceError`, `ConflictError`) wires a human-readable `code`, HTTP `status_code`, optional `field`, and structured `details`. Legacy wrappers (`OrderNotFoundError`, `InvalidStatusTransitionError`, `TeamsNotificationError`) preserve historical naming while still inheriting the same payload shape.
- **Idempotency guard:** `check_recent_notification` queries `TeamsNotification` rows marked `NotificationStatus.SENT` and only returns a hit if the last successful notification for an `order_id` occurred within the configured window (default 60 seconds). Calling code can skip duplicate Teams/Power Automate pushes when this helper returns a record.
- **Webhook security:** `verify_webhook_signature` normalizes every flavor of HMAC-SHA256 signature (hex with/without `sha256=` prefix, space-delimited, base64 variants) and accepts either raw or base64-encoded shared secrets (including Stripe-style `whsec_` tokens). It logs failures, gracefully allows development when no secret is configured, and exposes a single boolean that `InflowService.verify_webhook_signature` and the `/api/inflow/webhook` route consume.
- **PDF helpers:** `wrap_text` delegates to ReportLab (`stringWidth`/`canvas.stringWidth`), `check_page_break` handles page overflow and `pdf.showPage()`, and `filter_picklines` subtracts already shipped quantities/serial numbers from incoming pick lines so picklists only print what remains unsent.
- **Building mapper:** Combines regex-first heuristics (common building codes, name patterns, parenthetical tokens, explicit addresses) with a daily-cached ArcGIS lookup (`get_building_data`, `match_address_to_building`). Normalization (uppercase, whitespace tightening, abbreviation expansion) and `COMMON_BUILDING_CODES` guard what counts as a valid match. Exported helpers (`extract_building_code_from_location`, `get_building_code_from_address`, `get_building_abbreviation`) provide fallbacks for location remarks, alternate addresses, or full shipping addresses.
- **Timezone helpers:** `get_cst_datetime`, `get_date_in_cst`, and `is_morning_in_cst` wrap `datetime` conversion into Central Standard Time (UTC-6) with consistent formatting so scheduling logic always operates in the same timezone regardless of where the request originated.

- **Broadcast deduplication:** `BroadcastDeduplicator` (singleton `broadcast_dedup`) collapses rapid, repeated Socket.IO broadcast requests into a single call after a configurable cooldown (default 2s). Uses `threading.Timer` internally — if a new request arrives while a timer is pending, the timer resets so only one broadcast fires after the burst subsides. Imported by routes (`orders.py`, `delivery_runs.py`, `inflow.py`, `vehicle_checkouts.py`) and `scheduler.py` instead of raw `threading.Thread` spawning.
- **CSRF protection:** `csrf_protect` decorator verifies `Origin`/`Referer` headers against allowed CORS origins on state-changing endpoints. Applied to auth write endpoints (logout, session revocation). Works as a defense-in-depth layer alongside `SameSite=Lax` session cookies.

## Flow

- Inflow webhooks arrive at `backend/app/api/routes/inflow.py`, which reads headers, grabs every active `InflowWebhook.secret`, and delegates signature validation to `app.services.inflow_service.InflowService.verify_webhook_signature`. That service calls `app.utils.webhook_security.verify_webhook_signature`, passing the raw payload, header value, and app configuration (`settings.inflow_webhook_secret`).
- Once the webhook is trusted, `OrderService.create_order_from_inflow` and `LocationResolverService.resolve_location` rely on `app.utils.building_mapper` to coax building codes out of remarks, address parts, or ArcGIS/address lookups before writing or updating `Order` rows and logging the source (`remarks`, `address`, `arcgis`).
- Picklist generation (via `PicklistService`, `OrderService.generate_picklist`, and `PdfService`) uses `app.utils.pdf_helpers.wrap_text`, `check_page_break`, and `filter_picklines` to layout ReportLab PDFs and avoid duplicate ship lines. Tests in `backend/tests/test_picklist_service.py` cover the same helpers, ensuring wrap counts and filtering remain stable.
- Service/business-layer methods raise `DNSApiError` subclasses whenever validation, status transitions, missing resources, file uploads, or external calls fail; `backend/app/api/middleware.py` catches these, turns them into `ErrorResponse`, and the API routes simply forward the structured JSON with the attached HTTP code.
- `DeliveryRunService` and other scheduling helpers import `app.utils.timezone` so they can name runs, gate morning vs. afternoon logic, and persist CST-based timestamps even when the database stores UTC. `get_cst_datetime` underpins run naming logic, while `is_morning_in_cst`/`get_date_in_cst` provide consistent labels and queries.
- Notification paths consult `app.utils.idempotency.check_recent_notification` (which looks at `TeamsNotification` + `NotificationStatus.SENT`) before enqueueing Teams recipients or other message queues so short-lived retries do not flood the SharePoint/Power Automate flow.

## Integration

- `backend/app/api/middleware.py` and every route under `backend/app/api/routes/*` (orders, delivery runs, vehicle checkouts, auth, Inflow) raise and catch `DNSApiError` subclasses from `backend/app/utils/exceptions.py`, ensuring clients always receive a `code/message/status`. Services (`OrderService`, `PrintJobService`, `DeliveryRunService`, `VehicleCheckoutService`, `SamlAuthService`) import the same exceptions to keep control flow uniform.
- `backend/app/services/inflow_service.py` and `backend/app/api/routes/inflow.py` depend on `app.utils.webhook_security.verify_webhook_signature` to gate incoming payloads; the former exposes a wrapper that the route calls once per secret stored in `InflowWebhook`.
- `app.services.picklist_service.PicklistService`, `app.services.pdf_service.PdfService`, `app.services.order_service.OrderService`, and their tests call `app.utils.pdf_helpers.wrap_text`, `check_page_break`, and `filter_picklines` to prep ReportLab-based PDFs and ensure only unshipped quantities are rendered.
- `app.services.location_resolver_service.LocationResolverService`, `backend/scripts/database_manager.py`, and `app.services.order_service.OrderService` also depend on `app.utils.building_mapper` for building-code resolution; the location resolver layers heuristics atop `extract_building_code_from_location` and ArcGIS caching helpers before the order creation pipeline persists a canonical building reference.
- Scheduling/route helpers like `app.services.delivery_run_service.DeliveryRunService` import `app.utils.timezone` so naming, filtering, and morning/afternoon detection stay in CST even though `DeliveryRun.created_at` is stored in UTC.
- `app.utils.idempotency.check_recent_notification` reads `app.models.TeamsNotification` and `NotificationStatus.SENT` within a sliding window to avoid duplicate Teams/SharePoint notifications; documentation and the `TeamsNotification` codemap describe how `TeamsRecipientService` and future background workers can call this helper before writing new queue files.
